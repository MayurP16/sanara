# Security Policy

## Supported Versions

Before the first public release, security fixes are applied on `main`.

After public releases begin, this policy can be updated to describe supported release lines more precisely.

## Reporting a Vulnerability

Do not open public issues for vulnerabilities.

Please report vulnerabilities through GitHub Security Advisories or by email at `sanara.lab@gmail.com`.

When possible, include:

- affected version or tag
- reproduction steps
- impact assessment
- suggested remediation

## Scope Notes

- Do not include real secrets or customer-sensitive data in reports.
- If credentials may have been exposed, rotate them immediately before or alongside reporting.
