# Support

## Getting help

For usage questions, bugs, or feature requests:

1. Check [README.md](README.md) and the public docs.
2. Review [docs/debugging.md](docs/debugging.md) for troubleshooting and artifact inspection.
3. Open a GitHub issue if you still need help.

If GitHub Discussions are enabled later, they can be added here for general questions.

## What to include in an issue

Helpful details include:

- Sanara version or tag
- the relevant workflow snippet
- the policy file, if it affects the behavior
- the Terraform target layout, if relevant
- important artifact excerpts such as:
  - `artifacts/runlog.jsonl`
  - `artifacts/run_summary.json`
  - `artifacts/rescan/targeted_results_final.json`

Do not include secrets, credentials, or sensitive infrastructure data.

## Security issues

For vulnerabilities, do not open a public issue.

Use the reporting process in [SECURITY.md](SECURITY.md).
